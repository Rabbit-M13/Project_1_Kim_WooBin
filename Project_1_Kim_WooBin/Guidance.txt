<미니 벅샷 룰렛>

- 3개의 레벨 구성
- 아이템(아이템 별 최고 3개 소지 가능)
    - 회복약(Beer), 안경(다음 탄창 판별 - VisionGlasses), 공격력 증가 아이템(GunPowder), 골드 획득량 증가 아이템(GoldSack)
- 게임 구성
    - 라운드 별 라이프 포인트 상이 :  3, 4, 5 || 고정값
    - 라운드 별 제공되는 탄약 수 상이 :  5, 6, 7 ⇒ 개수 중에서 비율을 랜덤으로? || 
    - 라운드의 구성
        1. 행동 선택(1. 상대에게 쏜다 / 2. 나에게 쏜다 / 3. 아이템을 사용한다)
        2. 상대에게 발사시 라이프 포인트 조정 후 상대 턴으로 넘어감
        3. 나에게 발사시 피격 여부에 따라 체력 조정 후 상태 턴 or 다시 나의 턴
    
    ⇒ BattleShip의 JustSunk()처럼 bool JustGotShot();으로 턴 교체 여부 결정 예정
    
    ⇒ 상대방은 1/3확률로 본인에게 발포, 2/3확률로 플레이어에게 발포
    
    - 라운드의 종료 조건은 플레이어와 적의 체력 소진 여부 (단일 조건)
    - 탄창은 라운드 별 공포탄, 실탄의 비율이 다르게 설정
    - 라운드 종료 전 탄창이 모두 소진되면 새롭게 리셋됨
    - 라운드 종료 후 구매할 수 있는 상점 라운드 진행
- 구현에 필요한 요소들 예상 목록
    - GameManager
    - Actor class
        
        ⇒ Player
        
        ⇒ Enemy
        
        1. 라이프 포인트
        2. JustGotShot(?) ⇒ BattleShip코드 다시 확인해보기
    - Item class
        - HP drink : 체력을 1 올려준다.
        - 돋보기 : 총에 장전되어 있는 다음 탄창의 종류를 알 수 있다.
        - 톱 : 공격력 증가
        - 골드 주머니 : 획득하는 골드량 50% 증가
    - RoundSystem class
        - Monster class와 비슷하게 해야하지 않을까?
        - CurrentRound = 1, 2, 3
        - MaxMagNumber = 5, 6, 7
    - Store
        - Item class와 상호작용해야 함
